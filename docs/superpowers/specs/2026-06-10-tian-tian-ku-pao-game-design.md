# 天天酷跑 HTML 小游戏 — 设计规格

> 一个类"天天酷跑"的横版无尽跑酷 HTML 游戏，纯 Canvas 渲染，单文件部署。

## 1. 核心玩法

- **自动奔跑**：角色从左向右自动前进，游戏世界（地面、障碍、金币）向左滚动
- **跳跃（空格键）**：越过地面障碍和坑洞
- **下滑（下箭头键）**：躲避空中障碍（飞鸟）
- **目标**：跑尽可能远的距离，收集尽可能多的金币

## 2. 游戏流程

```
开始画面 → 游戏中 → 死亡 → 结算画面 →（循环回开始画面）
```

- **开始画面**：游戏标题、像素角色展示、操作说明（空格=跳跃，↓=下滑）、"按空格开始"
- **游戏中**：HUD 显示距离和金币数，实时更新
- **死亡**：角色碰撞动画（弹飞/碎裂），短暂延迟后进入结算
- **结算画面**：本次距离、本次金币、历史最高距离（localStorage 持久化）、"按空格重新开始"

## 3. 视觉风格 — 像素风 Q 版

| 元素 | 描述 |
|------|------|
| **角色** | 像素数据数组驱动，Canvas 逐像素绘制。跑步循环 3 帧，跳跃帧，下滑帧 |
| **地面** | 绿色像素块拼接，带纹理变化 |
| **背景** | 两层视差滚动（远山 + 云朵），深蓝到浅蓝渐变天空 |
| **障碍物** | 地面柱（棕色渐变）、空中飞鸟（红色像素鸟）、坑洞（黑色）、移动障碍（紫色悬浮方块） |
| **金币** | 金色方块，带旋转缩放动画 + 闪光效果 |
| **UI** | 像素风格字体（使用 `Press Start 2P` 或 `VT323` 等 Web 字体，或手工绘制像素字） |

### 角色像素数据

角色大小约 16×16 像素，在 Canvas 上以 3× 或 4× 放大渲染（48×48 或 64×64 显示尺寸）。定义 RGB 三元组数组作为像素映射：

- **跑步帧 1/2/3**：左右腿交替
- **跳跃帧**：四肢张开
- **下滑帧**：扁平姿态，高度减半

## 4. 场景系统

### 4.1 背景层

- 天空渐变（从顶部的浅蓝到底部的深蓝，静止）
- 第 1 视差层：云朵（大而慢，远景，滚动速度 = 游戏速度 × 0.2）
- 第 2 视差层：远山/建筑轮廓（滚动速度 = 游戏速度 × 0.5）

### 4.2 地面

- 从底部往上 80px 高度
- 绿色/棕色交替像素块
- 持续向左滚动，右侧不断生成新块

### 4.3 坑洞

- 地面出现缺口（黑色深渊）
- 需要跳跃才能通过
- 掉落即死亡

## 5. 障碍物系统

### 5.1 类型

| 类型 | 出现时机 | 应对方式 | 尺寸（显示像素） |
|------|----------|----------|------------------|
| 地面柱 (Stump) | 从一开始 | 跳跃 | 24×40 |
| 空中飞鸟 (Bird) | 从一开始 | 下滑 | 20×16 |
| 坑洞 (Pit) | 5秒后 | 跳跃 | 48×48（缺口） |
| 移动障碍 (Mover) | 20秒后 | 根据位置跳跃/下滑 | 24×24，上下正弦运动 |

### 5.2 生成规则

- 每 1.0～2.5 秒（随机间隔）生成一个障碍物
- 障碍物类型随机选择（根据当前可用类型等概率）
- 同一时刻只生成一个障碍
- 最小间隔 0.8 秒，保证玩家有反应时间
- 坑洞宽度固定为 48px，表现为地面段缺失

### 5.3 难度曲线

- 起始速度：6 像素/帧（≈360 px/s，60fps）
- 加速速率：每 10 秒 +0.5 px/f
- 最大速度：20 px/f
- 障碍生成频率同步加速：最小间隔从 1.5 秒逐步缩短至 0.8 秒

## 6. 角色系统

### 6.1 状态

```
IDLE → RUNNING → JUMPING → FALLING → SLIDING → DEAD
                 ↑_________________________↓
                    （JUMPING 和 SLIDING 结束后回到 RUNNING）
```

- **IDLE**：开始画面中的待机姿态（呼吸/眨眼动画）
- **RUNNING**：默认状态，3 帧跑步循环
- **JUMPING**：按下空格后进入，固定上升弧线 + 下落
- **FALLING**：跳跃顶点后的下落段
- **SLIDING**：按下 ↓ 后进入，持续按住则保持，松开回到 RUNNING
- **DEAD**：碰撞后的短动画

### 6.2 物理参数

- 跳跃：初速度 -9 px/f（上升），重力 0.5 px/f²
- 跳跃持续时间：约 36 帧（0.6 秒）
- 跳跃高度：约 81 px（显示距离）
- 下滑持续时间：按下即生效，持续至松开
- 下滑时碰撞盒高度减半（碰不到飞鸟，但会被地面障碍碰到）

## 7. 金币系统

- 金币与障碍物独立生成
- 每 1.5～3.0 秒随机在跑道区域生成一串（1～3 个）金币
- 高度：地面之上 20～120px 随机位置
- 金币动画：大小脉冲（0.8x～1.2x）+ 金色闪光闪烁
- 收集判定：角色碰撞盒与金币碰撞盒重叠
- 收集效果：金币消失 + 5 个粒子迸发（金色小方块向四周飞散并渐隐）
- 每个金币 +1 计数

## 8. HUD

- 左上角：**距离**（米，整数，每移动一段距离+1）
- 右上角：**金币数**（带金币小图标）
- 像素风格，白色描边文字，确保在任何背景上都清晰

## 9. 碰撞检测

采用 **AABB（轴对齐包围盒）碰撞**：

- 角色碰撞盒：以角色中心为基准，略小于视觉尺寸（宽度的 70%、高度的 80%），让手感更好（宽容判定）
- 障碍物碰撞盒：与视觉尺寸一致
- 坑洞碰撞：当角色 X 位置落入坑洞范围且 Y 位置在地面高度时判定死亡
- 每帧检测一次

## 10. 得分系统

- **距离分**：每帧根据当前速度累加，每累积 100px = 1 米
- **金币**：每个独立计数
- 结算时显示：距离（米）、金币数、总分（距离 + 金币×10）
- 最高分（最高距离）通过 localStorage 持久化

## 11. 死亡与结算

- 碰撞触发 → 角色 DEAD 动画（弹飞旋转，约 0.5 秒）
- 屏幕闪红 2 帧
- 结算画面淡入
- 显示：本次信息 + 最高记录对比
- 按空格回到开始画面

## 12. 技术栈

| 方面 | 选择 |
|------|------|
| **渲染** | Canvas 2D API，60fps requestAnimationFrame |
| **结构** | 单 HTML 文件，所有 CSS 内联 `<style>`，所有 JS 内联 `<script>` |
| **无外部依赖** | 不使用任何 npm 包或 CDN 库 |
| **字体** | Google Fonts CDN 引用 `Press Start 2P`（像素风） |
| **存储** | localStorage（最高分） |
| **音效** | 无 |

## 13. 文件结构

```
project_1/
└── index.html          # 单文件游戏，包含所有 HTML/CSS/JS
```

所有代码在一个文件中，双击即可游玩。

## 14. 代码架构（JavaScript 模块划分）

所有 JS 代码在 `index.html` 的 `<script>` 标签内，按以下逻辑分区（使用 IIFE 或对象命名空间隔离）：

```
Game               # 主控制器：初始化、游戏循环、状态管理
├── Renderer       # 所有 Canvas 绘制逻辑
│   ├── drawBackground()   # 天空、视差层
│   ├── drawGround()       # 地面
│   ├── drawPlayer()       # 角色（根据当前状态/帧）
│   ├── drawObstacles()    # 所有障碍物
│   ├── drawCoins()        # 所有金币
│   └── drawHUD()          # 距离 + 金币数
├── Player         # 角色状态、物理、动画帧
├── ObstacleManager # 障碍物生成、更新、回收
├── CoinManager    # 金币生成、更新、收集
├── Collision      # AABB 碰撞检测
├── ParticleSystem # 粒子特效（金币收集迸发、死亡效果）
└── UI             # 开始画面、结算画面覆盖层
```
